# Product Requirements Document: Tessera

## 1. Executive Summary
Tessera is an opinionated, local-first AI desktop orchestrator designed for business users. It rejects the fragile, open-ended "vibe coding" of standard AI chatbots. Instead, Tessera utilizes a highly deterministic state-machine architecture to execute complex business workflows (e.g., CRM updates, lead syncing, data extraction) with absolute predictability. It pairs a secure, lightweight OS shell with a robust local memory network to guarantee strict operational boundaries, data privacy, and verifiable Human-in-the-Loop execution.

## 2. Target Audience
* **Business Operators & SaaS Founders:** Users who require reliable automation for daily operations without trusting sensitive client data to fully autonomous cloud agents.
* **Privacy-Conscious Professionals:** Consultants and managers who need fully air-gapped or hybrid-local execution capabilities.
* **Workflow Power Users:** Individuals frustrated by having to repeat complex conversational prompts to achieve the same result twice.

## 3. Core Architecture & Stack
Tessera operates on a "Sidecar & CLI" architecture to maintain strict physical and memory boundaries:
* **The Shell:** Tauri v2 (Rust-based, capability-scoped OS boundary).
* **The Orchestrator:** Bun (High-performance sidecar daemon).
* **The State Engine:** XState (Deterministic DAG workflow execution and serialization).
* **The Cognitive Loop:** `@badlogic/pi-agent-core` (Minimalist tool-calling loop).
* **The Tooling:** Pre-compiled standalone CLI binaries (e.g., `workspace-cli`) acting as the only execution levers.
* **Inference Layer:** Vercel AI SDK acting as a Provider Abstraction Layer (Supporting OpenRouter BYOK and Local BYOR like Ollama).

## 4. Key Capabilities & Product Requirements

### 4.1 Deterministic Workflow Execution
* The system must execute workflows defined strictly as XState JSON/TypeScript state machines.
* The LLM must never dictate the workflow path; it is strictly invoked by XState to process data or select CLI parameters.
* All tool arguments generated by the AI must pass strict Zod schema validation before execution.

### 4.2 The Dual-Memory System
* **Hindsight (Episodic):** An embedded PostgreSQL/`pgvector` daemon running locally without Docker. It stores immutable facts and temporal workflow outcomes for immediate context retrieval (Pre-Flight).
* **GBrain (Semantic):** A background compiler that uses idle system time to synthesize Hindsight facts into a structured, cross-referenced Markdown wiki stored in a local, Git-backed directory.

### 4.3 Human-in-the-Loop (HITL) Security
* The orchestrator must classify all CLI tools strictly as "Read" (Safe) or "Write" (Destructive).
* Upon reaching a "Write" action, XState must suspend execution indefinitely and trigger a native Tauri OS-level UI dialog displaying the exact payload.
* Execution cannot proceed without explicit boolean approval from the user.

### 4.4 The Auto-Compiler ("Record Macro")
* Users must be able to convert a successful open-ended chat session into a repeatable XState workflow.
* A dedicated reasoning model parses the Hindsight session trace, generates the XState JSON, validates it against a strict schema in a background loop, and dynamically registers the new workflow in the UI command palette.

### 4.5 Built-In CLI ERP/CRM
* Tessera must ship with a lightweight, SQLite-backed internal CRM/ERP accessed via CLI tools to provide immediate value out of the box, reducing the friction of configuring external OAuth integrations on day one.

## 5. Implementation Phases
To prevent AI coding assistants from tangling cross-process boundaries, development must be strictly siloed into these consecutive phases. 

### Phase 1: The Secure Sandbox (OS Boundary)
**Goal:** Establish secure, isolated communication without any AI or LLM logic.
* Initialize the Tauri v2 shell and the React webview.
* Build the Bun sidecar gateway.
* Define the `capabilities.json` explicitly allowing Bun to spawn child processes.
* Create a dummy `workspace-cli` binary. Prove that clicking a button in the UI tells Bun to spawn the CLI, capture stdout, and return the payload securely.

### Phase 2: The Cognitive Engine (Inference & Tooling)
**Goal:** Build the isolated AI execution loop in the terminal backend.
* Integrate the Vercel AI SDK Provider Abstraction Layer to support both OpenRouter (Cloud) and standard localhost polling (Ollama/LM Studio).
* Implement `@badlogic/pi-agent-core` within Bun.
* Implement the hardcoded Read/Write capability intercept in Bun (without the UI hook yet).

### Phase 3: Deterministic Orchestration (XState)
**Goal:** Remove open-ended planning by forcing the agent into rigid workflows.
* Integrate XState into the Bun sidecar.
* Define the first manual YAML/JSON workflow manifest.
* Wire the HITL intercept: Configure XState to suspend on "Write" tools, send the exact payload to Tauri for the approval UI, and resume execution upon receiving the user's explicit confirmation.
* Implement SQLite state checkpointing so paused XState workflows survive application reboots.

### Phase 4: The Dual-Memory Integration
**Goal:** Ground the agent in local reality.
* Configure the Tauri installer/boot sequence to manage the `hindsight-local-mcp` embedded Postgres daemon.
* Wire the `Retain` hook (saving XState outcomes to Hindsight) and the `Recall` hook (injecting operational facts into Pi Agent before Step 1).
* Configure the Tauri idle-time detector to trigger the background GBrain wiki compilation script.

### Phase 5: Extensibility & The Auto-Compiler
**Goal:** Make the platform dynamically grow based on user actions.
* Implement `react-jsonschema-form` in the Tauri UI to auto-generate input forms directly from uploaded workflow manifests.
* Build the Auto-Compiler loop: Extract a successful session trace from Hindsight, route it to an OpenRouter reasoning model, auto-validate the generated XState JSON, and hot-reload the Tauri command palette.

### Phase 6: The built-in SQLite CLI CRM/ERP
**Goal:** Build the built-in SQLite CLI CRM and map its commands as Zod-validated skills for Pi Agent.
* Initialize a new Tauri project named `tessera-cli`.
* Install Bun, SQLite CLI, and the `sqlite3-cli` package (or similar utility for running SQLite commands programmatically).
* Implement the database schema for the CRM/ERP (e.g., contacts, leads, tasks, sales data).
* Create the SQLite CLI commands for managing CRM data.
* Integrate the SQLite CLI commands into the Bun sidecar as executable tools.
* Map these commands as Zod-validated skills for Pi Agent.
