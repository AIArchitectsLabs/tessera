The SEO and GEO article recipe is a reference pattern for content workflows. It is useful because it exercises intake normalization, bounded research planning, read-only source collection, structured writing, review, rework, and final materialization.

## Reusable Pattern

1. Normalize loose user intake into a strict artifact.
2. Derive domain-specific research work items.
3. Fan out through declared read-only capabilities.
4. Normalize noisy source outputs before fan-in.
5. Prepare compact writing seeds for agent prompts.
6. Store raw agent output first, then structure it deterministically.
7. Review, rework, and pause for human approval.
8. Write only durable final artifacts.

## What To Rewrite

When adapting this recipe, rewrite the schemas, prompts, taxonomies, scoring rules, source plans, fixtures, and final templates for the new domain. Keep the graph choreography only when the same workflow shape still fits the business outcome.
