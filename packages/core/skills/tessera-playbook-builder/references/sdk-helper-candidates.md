# SDK Helper Candidates

Use this reference to record Phase 2B candidates. Do not implement SDK helpers during Phase 2A.5.

## Promotion Standard

Promote a helper only when it has:

- repeated evidence from SEO/GEO, supply-chain, or a forward test
- a domain-neutral API shape
- clear reduction in boilerplate or validation failures
- tests that preserve validator behavior

## Candidate Template

```markdown
## Candidate: <name>

Evidence:

- <path or command>

Repeated friction:

- <what authors repeated or got wrong>

Domain-neutral API shape:

- <candidate API without domain vocabulary>

Decision:

- promote in 2B | defer | reject as domain-specific | needs more evidence
```
