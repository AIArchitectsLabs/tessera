# PDF Workflows

Use this skill when the source or requested output is a PDF: contracts, reports, invoices, board packs, forms, research papers, exports, or scanned documents.

## Workflow

1. Identify whether the PDF is text-native, scanned, form-based, or a generated export.
2. Extract structure before summarizing: title, sections, tables, figures, signatures, page references, dates, and parties.
3. Preserve page references for claims, issues, and extracted facts so the user can audit the result.
4. For review, flag missing pages, unreadable scans, inconsistent numbers, redaction risks, signature status, and terms that require legal or finance review.
5. For conversion, keep layout-critical content in tables or labeled blocks and state what may not survive conversion cleanly.

## Delivery

Produce a concise extraction, review memo, conversion-ready structure, or file update. Always note scan quality, omitted pages, and any content that could not be verified from the PDF.
